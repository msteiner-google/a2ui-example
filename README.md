# A2UI Example Project

This repository demonstrates a full-stack application integrating **Agentic Design Kit (ADK)**, **Agent-to-Agent (A2A)** protocol, and **A2UI** for interactive, AI-driven user interfaces. It features a RAG (Retrieval-Augmented Generation) agent that helps employees find insurance policies using a modern, interactive chat interface.

## 🏗️ Architecture Overview

The project is divided into two main components:
- **Backend (Python)**: A FastAPI-based server using ADK and A2A to manage LLM agents and sub-agents.
- **Frontend (Angular)**: A responsive chat application using `@a2ui/angular` to render interactive widgets dynamically generated by the backend.

### Backend Components
- **`src/adk2/main.py`**: Entry point that bootstraps the A2A server and agent card.
- **`src/adk2/agent_executor.py`**: Handles incoming A2A messages, maps UI events to agent queries, and manages session state.
- **`src/adk2/agent.py`**: Defines the root agent (`ui_demo_assistant`) which delegates tasks to specialized sub-agents.
- **`src/adk2/subagents/rag_agent.py`**: A specialized sub-agent that performs fuzzy search across a mock policy database and returns A2UI JSON payloads for interactive selection and filtering.
- **`data/`**: Contains mock data (`mock_policies.json`) and Jinja2 templates for A2UI widgets.

### Frontend Components (`renderer/`)
- **Angular v18+**: Uses standalone components and signals for state management.
- **`chat.service.ts`**: Communicates with the backend via JSON-RPC, processes A2UI messages, and manages the chat history.
- **`a2ui-surface`**: Dynamically renders UI components (cards, dropdowns, buttons) based on server responses.

## 🚀 Getting Started

### Prerequisites
- Python 3.14+
- Node.js & npm
- [uv](https://github.com/astral-sh/uv) (recommended for Python dependency management)

### Backend Setup
1. Install dependencies:
   ```bash
   uv sync
   ```
2. Start the agent server:
   ```bash
   uv run adk2
   ```
   The server will start on `http://localhost:8080`.

### Frontend Setup
1. Navigate to the renderer directory:
   ```bash
   cd renderer
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the Angular dev server:
   ```bash
   npm start
   ```
   The UI will be available at `http://localhost:4200`. It is configured to proxy `/rpc` requests to the backend server.

## 💡 How It Works

1. **Querying**: When a user asks about policies, the `rag_agent` extracts the query and searches the mock database.
2. **Interactive UI**: If multiple policies are found, the agent renders a `show_dropdown.json.j2` template. This generates an A2UI widget with a searchable dropdown and a "Filter" text field.
3. **Dynamic Appending**: To ensure a seamless chat history, each UI response is generated with a unique `surface_id`. This causes the frontend to append a new widget to the chat bubble instead of overwriting previous ones.
4. **Filtering**: The "Filter" button sends a `filter_policies` event back to the server, which triggers a new search and appends a refined result widget.

