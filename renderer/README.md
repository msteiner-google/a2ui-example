# A2UI v0.8 Chat Renderer

This project is a functional chat application built with **Angular** that demonstrates the integration of the **A2UI (Agent-to-User Interface)** renderer using the **v0.8 specification**.

## 🚀 Overview

The application allows users to interact with an AI agent through a traditional chat interface. Unlike standard chat apps, this renderer can process and display interactive UI components (surfaces) generated by the agent on the fly, such as forms, dropdowns, and layouts.

### Key Technical Features
- **A2UI v0.8 Protocol**: Strictly adheres to the version 0.8 specification, using flat component lists and streaming-ready message types (`surfaceUpdate`, `dataModelUpdate`).
- **Angular Signals**: Leverages Angular's reactive system for fine-grained, high-performance UI updates.
- **JSON-RPC 2.0**: Communicates with the backend using a structured JSON-RPC over HTTP protocol.
- **Interactive Surfaces**: Supports multi-surface rendering where agent-dispatched actions (like button clicks) are captured and sent back to the server.
- **Dev Proxy**: Includes a pre-configured Angular proxy (`proxy.conf.json`) to handle CORS and forward requests seamlessly to a local backend on port 8080.

## 🛠 Project Structure

- **`ChatService`**: The core logic handler. It manages the chat history, handles JSON-RPC communication, and bridges the server messages to the A2UI `MessageProcessor`.
- **`A2uiRenderer`**: Uses the `@a2ui/angular/v0_8` library to map abstract component definitions (like `Button` or `Column`) to real Angular components.
- **`app.config.ts`**: Configures the A2UI environment, registers the standard component catalog, and provides a custom (no-op) theme to satisfy v0.8 requirements.

## 🚦 Getting Started

### 1. Prerequisites
- Node.js (v18+)
- A backend server running on `http://localhost:8080` that implements the JSON-RPC `message/send` method.

### 2. Installation
```bash
npm install
```

### 3. Development Server
Run the following command to start the app with the proxy enabled:
```bash
npx ng serve
```
Navigate to `http://localhost:4200/`.

## 💬 How to Interact

1. **Text Messages**: Type a message in the input box. It is sent as a `text` part in a `message/send` RPC call.
2. **UI Surfaces**: If the agent returns an A2UI payload, a "surface" will appear in the chat.
3. **Interactions**: Click buttons or fill out fields within a surface. These actions are automatically captured by the `MessageProcessor` and dispatched back to the server as `data` parts, allowing the agent to respond to specific UI events.
4. **Visual Feedback**: A spinner appears whenever a request is pending, ensuring you know when the agent is "thinking."

## 📜 Protocol Details (v0.8)

This app implements the following server-to-client message types:
- `beginRendering`: Initializes a surface area.
- `surfaceUpdate`: Upserts components into the surface's component tree.
- `dataModelUpdate`: Updates the state/data bound to those components.
- `deleteSurface`: Removes a surface from the UI.
